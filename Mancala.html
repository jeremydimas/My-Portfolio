Mancala in C++

Objectives

The objective of this class project is to write a program that plays the game Mancala well. Several classes need to be written to build and play the game. Some of these classes will already be speciifed and public member functions must be implemented. These classes are Board, Player, HumanPlayer, BadPlayer, SmartPlayer, and Game.

Skills

Throughout the course of the project, knowledge of recursion, sorting, trees, linked lists, memory allocation, stacks, queues, and pointers is tested. Critical thinking and data analysis will be used to develop the class SmartPlayer, an AI model that will make winning the game challenging.

How does the game work?

The game consists of a North side (Homer) and South side (Marge) with North's pot being on the left side of the screen and South's pot being on the right side of the screen. South side is player one and will always go first, while North side is player two and will follow after. Marge chooses to move her beans from pot one and Homer chooses to move his beans from pot one as well. Every time someone chooses to make a move, a new display will be generated after a turn is completed (courtesy of the Game class), hence there are three displays of the board. (The game is played going clockwise direction.)

Marge decides to move her beans from pot two but notice right after her turn is completed, Homer was able to have four turns completed total before Marge's turn. This is because the last bean that was placed in each of Homer's turns landed last in the pot, therefore he is able to receive another turn. Ideally when playing an actual person you'd hope the person you're playing against doesn't have this level of IQ, but because of the SmartPlayer class and the algorithm that was created for this game, the AI model (North side) already has this intuition and will make these attempts whenever possible.

Marge can only move her beans from pot three. Homer completes his attempts and the turn goes back to Marge, however since Marge no longer has any more beans to move (sow) the beans leftover automatically get transferred over to the pertaining side. Homer obtained the most beans with 10, therefore Homer wins!

You can't "outsmart" the game

The game will only accept valid inputs depending on the number of holes per side, in this case only 1,2, or 3.

In that case, right-click on the executable instead through your "Downloads" folder, select Open With and then Terminal.app. Select the gray Open button on the popup that again says it doesn't know the developer and the game will be ready to play. (This might not be yet accessible on Windows yet, but feel free to try it in case it does work.) Have fun!
